Overview
--------

Requirements
^^^^^^^^^^^^

The command line interface is a set of Python_ based system-administration and 
deployment tools.

Check you have Python_ installed by typing::

    $ python --version
    Python 2.5.1

Additionally, Ice_ must be installed on your machine::

    $ python -c "import Ice"

The command line interface is currently bundled with the OMERO.server. 
Download the version corresponding to your system from the 
:omero_plone:`OMERO downloads <downloads>` page.

Once the server is downloaded, the command line interface is located under the 
``bin/`` subdirectory::

    $ cd OMERO.server
    $ bin/omero -h
    OMERO Python Shell. Version 4.4.5-ice33
    Type "help" for more information, "quit" or Ctrl-D to exit
    omero>

Command help
^^^^^^^^^^^^

The command line interface is divided into several commands which may 
themselves contain subcommands. You can investigate the various commands 
available using the `-h/--help` option::

    $ bin/omero -h

Again, we can use `-h` repeatedly to get more details on each of these
sub-commands::

    $ bin/omero admin -h
    $ bin/omero admin start -h

The `help` command can be used to get info on other commands or options::

    $ bin/omero help debug       # debug is an option
    $ bin/omero help admin       # same as bin/omero admin -h

Command line workflow
^^^^^^^^^^^^^^^^^^^^^

There are three ways to use the command line tools:

#.  By first explicitly logging into to the server use the :command:`login`
    command (see :ref:`sessions_login` for more details)::

        $ bin/omero login username@servername:4064

    Once logged in, by calling the desired command, e.g. the :command:`import` 
    command::

        $ bin/omero import image.tiff

#.  By passing the session arguments directly to the desired command, e.g.::

        $ bin/omero -s servername -u username -p 4064 import image.tiff

#.  By calling the desired command without login arguments. You will be asked
    to login::

        $ bin/omero import image.tiff
        Server: [servername]
        Username: [username]
        Password:

.. index::
    single: import

Import images
-------------

:command:`import` is probably the first command many will users want to use. 
Running the command without any arguments will produce a help listing::

    $ bin/omero import -h

So, to upload a file image.tiff, use the :command:`import` command as::

    $ bin/omero import image.tiff

Sessions
--------

:command:`sessions` creates and handles user sessions stored locally on disk. 
Several sessions can be active simultaneously, but only one will be used for a 
single invocation of :command:`bin/omero`::

    $ bin/omero sessions -h

.. index::
    single: sessions; login
 	single: sessions; file
    single: login

.. _sessions_login:

Create a session
^^^^^^^^^^^^^^^^

A session can be created using the :command:`login` command. The 
:command:`bin/omero login` command is a shortcut for :command:`bin/omero 
sessions login` subcommand

Without arguments, the interface will ask for the server name, user name and 
password to use::

    $ bin/omero login
    Previously logged in to localhost:4064 as root
    Server: [localhost]
    Username: [root]
    Password:

The server name, port and username can be passed to the :command:`login` 
command

#.  either directly within a connection string::

        $ bin/omero login username@servername:4064
        Password:

#.  or using the `-s/--server`, `-u/--user` and `-p/--port` optional 
    arguments::

        $ bin/omero login -s servername -u username -p 4064
        Password:


The session is stored locally on disk and its location can be retrieved using
the :command:`file` subcommand::

    $ bin/omero sessions file
    /Users/ome/omero/sessions/localhost/root/aec828e1-79bf-41f3-91e6-a4ac76ff1cd5

.. index::
    single: sessions; logout

Close a session
^^^^^^^^^^^^^^^

To terminate the current session, use the :command:`logout` subcommand::

    $ bin/omero logout

.. note::
    ``bin/omero logout`` is a  shortcut for ``bin/omero sessions logout``.

.. index::
    single: sessions; list

Multiple sessions
^^^^^^^^^^^^^^^^^

Stored sessions can be listed using the :command:`list` subcommand::

    $ bin/omero sessions list
     Server    | User | Group           | Session                              | Active    | Started
    -----------+------+-----------------+--------------------------------------+-----------+--------------------------
     localhost | test | read-annotate-2 | 22fccb8b-d04c-49ec-9d52-116a163728ca | Logged in | Fri Nov 23 14:55:25 2012
     localhost | root | system          | 1f800a16-1dc2-407a-8a85-fb44005306be | True      | Fri Nov 23 14:55:18 2012
    (2 rows)

Sessions keys can then be reused to switch between stored sessions::

    $ bin/omero sessions login -k 22fccb8b-d04c-49ec-9d52-116a163728ca
    Server: [localhost]
    Joined session 1f800a16-1dc2-407a-8a85-fb44005306be (root@localhost:4064).
    $ bin/omero sessions list
     Server    | User | Group           | Session                              | Active    | Started
    -----------+------+-----------------+--------------------------------------+-----------+--------------------------
     localhost | test | read-annotate-2 | 22fccb8b-d04c-49ec-9d52-116a163728ca | True      | Fri Nov 23 14:55:25 2012
     localhost | root | system          | 1f800a16-1dc2-407a-8a85-fb44005306be | Logged in | Fri Nov 23 14:55:18 2012
    (2 rows)

Sessions directory
^^^^^^^^^^^^^^^^^^

By default sessions are saved under ``~/omero/sessions``. If you want to use a
custom session directory, use the `--sessions-dir` optional argument in the 
:command:`sessions` subcommands::

    $ bin/omero login --sessions-dir=/tmp
    $ bin/omero sessions list --sessions-dir=/tmp
    $ bin/omero logout --sessions-dir=/tmp
